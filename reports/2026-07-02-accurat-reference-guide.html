SDRAM vs Flash

Critical — do not forget All register writes go to SDRAM (volatile). SAV.<axis> = <P1> copies to flash (non-volatile). At boot, RES auto-restores flash → SDRAM. P1 values: 0 = all, 3 = K registers, 5 = X registers, 7 = sequences. Forgetting to SAV after tuning is the #1 cause of "it worked yesterday" bugs.

3. Position Registers (Verified)

Previously: ML0 was called "Raw Position" and ML1 was "Setpoint Position". Now corrected to match manual §5.2.
RegisterManual NameDescriptionUnit
ML0Position setpointThe reference position the controller is commanding. This is the base value before the SET offset (ML4).incr
ML1Enhanced measured positionThe actual measured position from the encoder, after all processing (scale mapping, compensation, etc.). This is where the motor really is.incr
ML2Position tracking errorML0 − ML1. The instantaneous difference between setpoint and actual position.incr
ML4SET offsetThe offset applied by SET command. ML6 = ML0 + ML4.incr
ML6Adjusted reference positionML0 + ML4. The coordinate-system position used by the PID controller. This is what MVE moves to.m or t (LPI)
ML7Adjusted measured positionML1 + ML4. The actual position in the coordinate system. Used for stroke limit checking.m or t (LPI)
Key concept ML4 is the offset between the position setpoint and the coordinate system. When you execute SET.0 = 0L, ML4 is recalculated so that ML6 = 0. This is how you zero the coordinate system at the home position. ML0 never changes — only ML4 does.

4. Commissioning Pipeline

The 8-step hardware initialization sequence for commissioning an axis. Even if COMET automates most of this, understanding each step is essential for debugging.

#StepRegisterWhat it does
1Motor typeK241Linear (1) or rotary (2). Wrong setting = wrong commutation.
2EncoderK6002=1Vpp, 4=TTL, 5=EnDat 2.2, 16=EnDat 3. Requires SAV + reboot.
3CommutationK56Motor phase direction. AUT = 2 for auto-configure.
4PhasingK901=pulses (ironcore), 2=constant current, 7=small movement (ironless).
5Current limitsKF60, KF83KF60 = current ref limit (input), KF83 = phase overcurrent (output). Chain: KF83 ≥ KF60 ≥ KF61.
6I²t protectionKF84, KF85Motor thermal model. KF84 = time constant, KF85 = current level. Error 4'000 when tripped.
7PID tuningKF1, KF2, KF4P=immediate correction, I=steady-state, D=damping. KF5=anti-windup.
8HomingK400–1=end stop, 2–3=home switch, 8–11=ref mark, 22=immediate, 46=absolute encoder.

Current Limit Chain (Critical)

Must be correct The chain must be: KF83 ≥ KF60 ≥ KF61. If KF60 < KF61, the current reference limit clips before the force reference. If KF83 < KF60, phase overcurrent trips during normal operation.

7. Hardware Limit Switches

Configure external sensors (ToF, proximity, mechanical) as hardware limit switches. The controller treats these as hard stops — triggering causes immediate motor stop.

Setup Procedure

1
Set K58 = 1 to enable hardware limit switch I/O type (FDIN/DIN pins). Manual §8.7.3.1.
2
Set K33 = 1 to enable limit switch protection. Manual §8.7.3.
3
Assign positive limit to KL36 using bitmask (e.g., KL36:0 = 0x100000 for DIN5/X21_PIN_5). Subindex 0 = active high, 1 = active low. Manual §8.7.3.1.
4
Assign negative limit to KL37 using bitmask (same format as KL36). Manual §8.7.3.1.
5
Verify with M45: Bit 0 = positive limit triggered, Bit 1 = negative limit triggered, Bit 2 = home switch triggered. Manual §8.7.3.2.
6
SAV + reboot to persist.

Register Summary

RegisterNameDescription
K58Limit switch I/O type0=none, 1=FDIN/DIN, 2=encoder L1/L2, 3=encoder L/H
K33Enable protection0=disabled, 1=enabled
KL36Positive limit FDIN maskSubindex 0=active high, 1=active low
KL37Negative limit FDIN maskSubindex 0=active high, 1=active low
M45StatusBit 0=pos limit, Bit 1=neg limit, Bit 2=home switch
Upper limit not triggering? Common issue: upper limit assigned to KL37 (negative) instead of KL36 (positive). Check M45 bits when sensor is physically triggered. Verify subindex polarity (0 vs 1). Verify K32 (invert/swap encoder limits) is not interfering.

8. Sequence Programming

The ETEL Sequence language is a C subset. Key concepts for embedded engineers:

Entry Points

FunctionPurposeHow it runs
autostart()Boot entry pointRuns automatically at controller boot
func0() … func1023()Callable functionsJMP.<axis> = <n>, <thread>
error_handler0() … error_handler3()Error handlersAuto-jumped when error occurs on thread

Thread Rules

  • DUAL: 2 threads per axis (0, 1). QUAD: 1 thread per axis (0).
  • Threads are cooperative, not preemptive. yield() gives up CPU.
  • Always use yield() in polling loops — a loop without yield freezes the entire controller.
  • Local variables only at function start (before any statements). No inline declarations.
  • Arrays can only be global. No pointers. No structs. No recursion.

Data Types & Suffixes

TypeSizeSuffixExample
int32-bit(none)42
long64-bitL42L
float32-bitF3.14F
double64-bitD1.0D
Silent bug MVE.0 = 5.0; — the 5.0 is treated as integer increments, not ISO units. Must write 5.0D or 5.0F for ISO values. The controller moves, but to the wrong position.

Typical Drilling Sequence

#pragma linear_units ("mm, mm/s, mm/s²")
#define _X_ 0
#define _Y_ 1

double holes[][2] = {
    {10.0D, 20.5D}, {10.5D, 20.5D}, {11.0D, 20.5D},
};
#define NUM_HOLES 3

void drillCycle(void)
{
    for (int i = 0; i < NUM_HOLES; i++) {
        // 1. Move XY to hole position
        MVE.(_X_, _Y_) = iso2inc_l(holes[i][0], LPI_M),
                         iso2inc_l(holes[i][1], LPI_M);
        WTM.(_X_, _Y_);

        // 2. Plunge Z
        MVE.2 = iso2inc_l(-1.6D, LPI_M);
        WTM.2;

        // 3. Retract Z
        MVE.2 = iso2inc_l(5.0D, LPI_M);
        WTM.2;
    }
}

void autostart(void)
{
    RST.ALL;
    PWR.(_X_, _Y_, 2) = 1;
    IND.(_X_, _Y_, 2);
    WTM.(_X_, _Y_, 2);
    drillCycle();
}
Critical safety rule Never move X/Y while Z is still in the hole. Always WTM.2 after Z retract before moving X/Y. Skipping WTM = snapped drill bit and ruined board.

9. Error System

Three Severity Levels

LevelRegisterBehaviorKey Codes
Error M64 Stops motor, red LED, requires reset 2'000=overcurrent, 4'000=motor I²t, 11'000=controller I²t, 20'000=encoder, 24'000=overspeed, 65'000=stroke limit
Warning M66 Motor continues, must manually clear 4'000=I²t 50%, 23'000=tracking error >50% of KL30
Info M68 Auto-clears when resolved Various low-severity informational flags

Error Propagation

  • K140 = group membership (bitmask of which group each axis belongs to)
  • K141 = publish group mask (which groups this axis broadcasts errors to)
  • An error on one axis propagates to all axes in the same group
  • This is how a Z-axis error can stop X and Y — they share the same group
After error recovery RST clears the error but does not power the motor. Must explicitly call PWR.0 = 1. Check ML63 bit 36 before issuing any move. Re-home after error recovery — position may have been lost.

10. PID Control

PID Registers

RegisterNameRole
KF1Proportional gainImmediate response to position error. Higher = faster but more ringing.
KF2Integral gainEliminates steady-state error. Too high = slow oscillation (integrator windup).
KF4Derivative gainDamping, reduces overshoot. Too high = noise amplification.
KF5Anti-windupPrevents integrator saturation when output is capped.
KF61Force reference limitCaps PID output. Must be ≤ KF60.

Feedforward

RegisterNameRole
KF3Speed feedforwardAdds force proportional to target speed. Reduces tracking error during cruise phase.

Inside-Out Tuning

COMET5 has two layers: Current loop → Position loop (no velocity loop — it's inside the drive firmware).

Why inside-out Each outer loop's plant IS the closed-loop transfer function of the inner loop. If the current loop is oscillating, the position loop identification will produce garbage data (steady high-pitch noise instead of clean broadband response). Tune current loop first, then position loop.
